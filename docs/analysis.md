# Analysis

The `scripts/analysis/` directory contains utilities for comparing providers, generating plots, measuring variance, and tracking runs.

---

## Comparing runs

`scripts/analysis/compare_runs.py` computes statistics and generates comparison plots. Runs can differ by provider, model, or configuration:

```bash
uv run scripts/analysis/compare_runs.py \
  --output-dir results/images/ \
  --stats-output results/compare_stats.json
```

Produces:
- `results/images/scores_overall_weighted.png` — weighted average scores by metric across all plugins
- `results/images/scores_by_plugin.png` — per-plugin breakdown

You can restrict to specific runs or inference run IDs with `--runs` and `--inf-run-ids`.

---


## Tool call and latency plots

```bash
# Distribution of tool call counts
uv run scripts/analysis/plot_tool_calls.py results/answers_*.json --labels claude openai

# Latency distribution
uv run scripts/analysis/plot_latency.py results/answers_*.json --labels claude openai
```

Both accept `--output` to specify the output filename and `--stat` to control the statistic shown (e.g. `count`, `proportion`).

---

## Variance analysis

`scripts/analysis/variance_analysis.py` measures reproducibility across repeated runs:

```bash
# Inference variance (tool call count std, answer length std, answered rate)
uv run scripts/analysis/variance_analysis.py inference \
  results/answers_claude_panda.json results/answers_claude_falcon.json

# Eval variance (overall score std, per-tag std, per-assertion agreement)
uv run scripts/analysis/variance_analysis.py eval \
  results/eval_results_run1.json results/eval_results_run2.json
```

## Run tracking

`results/runs.db` is a SQLite database that records every inference and evaluation run. It is used for run deduplication (avoiding redundant re-runs when the dataset and config haven't changed) and for querying result history.

### Querying

```bash
# List all runs (one row per run)
uv run scripts/tracking/runs.py list

# Filter by provider or judge
uv run scripts/tracking/runs.py list --provider claude --judge gpt-4.1

# Export full flat table (one row per run × question) to CSV
uv run scripts/tracking/runs.py export --output results.csv

# Delete runs from the database (dry-run by default)
uv run scripts/tracking/runs.py delete --id 5
uv run scripts/tracking/runs.py delete --provider claude --yes --files  # also removes files
```

### Registering runs

If result files exist but are not yet in `runs.db` (e.g. after manually copying files), register them with:

```bash
# Auto-discover all pairs in results/
uv run scripts/tracking/register.py

# Register a specific pair
uv run scripts/tracking/register.py \
  results/answers_gpt-5.2_panda.json \
  results/eval_results_answers_gpt-5.2_panda_falcon.json
```
